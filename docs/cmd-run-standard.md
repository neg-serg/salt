# cmd.run Error Handling Standard

This document defines the mandatory error handling pattern for all `cmd.run` and `cmd.script` Salt states in this repository. Compliance is checked automatically by `tests/test_cmdrun_audit.py`.

## Required Elements

Every `cmd.run` or `cmd.script` state MUST include:

### 1. Shell Directive

Explicitly specify the shell interpreter. Never rely on the system default.

```yaml
my_state:
  cmd.run:
    - name: |
        echo "hello"
    - shell: /bin/bash    # or /bin/zsh
```

### 2. Idempotency Guard

Every command MUST have at least one guard to prevent re-execution (Constitution Principle I):

- `creates:` — when the command produces a known file
- `unless:` — when the result is a state change (package installed, config applied)
- `onlyif:` — to conditionally skip when dependencies are absent

```yaml
install_tool:
  cmd.run:
    - name: curl -fsSL https://example.com/tool | bash
    - creates: /usr/local/bin/tool
```

### 3. Error Handling

Multi-line shell commands MUST begin with `set -eo pipefail`:

```yaml
build_something:
  cmd.run:
    - name: |
        set -eo pipefail
        cd /tmp
        make && make install
    - shell: /bin/bash
    - creates: /usr/local/bin/something
```

Single-line commands inherit shell error behavior and don't need explicit `set -eo pipefail`.

### 4. Exception Annotation

If a command legitimately needs non-standard error handling (e.g., grep returning 1 for "not found"), annotate with a comment in the `name` field:

```yaml
check_if_present:
  cmd.run:
    - name: |
        # cmdrun:exception grep returns 1 when pattern not found
        grep -q "pattern" /etc/config || echo "not found"
    - shell: /bin/bash
    - unless: grep -q "pattern" /etc/config
```

The `# cmdrun:exception <reason>` annotation tells the audit test to skip the error handling check for this state.

## Compliance Levels

The audit test (`tests/test_cmdrun_audit.py`) checks three requirements per state:

| Requirement | Check | Severity |
|-------------|-------|----------|
| Shell directive | `shell:` key present | Error |
| Idempotency guard | `creates:`, `unless:`, or `onlyif:` present | Error |
| Error handling | `set -eo pipefail` in multi-line commands (or `cmdrun:exception`) | Warning |

## Scope

This standard applies to all `cmd.run` and `cmd.script` states, including those generated by macros. Macro-generated states are checked separately by `tests/test_macro_idempotency.py`.

All existing `cmd.run` states in the repo already follow this standard.
