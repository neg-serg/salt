# Настройка Video AI

Локальная генерация видео с помощью AI-моделей на AMD RX 7900 XTX (24 ГБ VRAM).

## Требования

- AMD RX 7900 XTX (24 ГБ VRAM, gfx1100)
- ROCm 6.3-6.4 (CachyOS поставляет совместимые пакеты)
- `/mnt/one` примонтирован, ~100 ГБ свободного места для полного набора моделей
- ffmpeg (установлен по умолчанию в CachyOS)

## Быстрый старт

1. Включите функцию и нужные модели в `states/data/hosts.yaml`:

   ```yaml
   features:
     video_ai: true
   ```

2. Включите модели в `states/data/video_ai.yaml`:

   ```yaml
   models:
     - id: ltx-23-distilled-fp8
       enabled: true  # лучшее качество, 4-8 шагов distilled
   ```

3. Примените:

   ```bash
   just apply video_ai
   ```

4. Сгенерируйте видео:

   ```bash
   gen-video "кошка гуляет по луне на закате"
   ```

## Активные модели

| Модель | VRAM | Скорость (прибл.) | Качество | Режимы |
|--------|------|-------------------|----------|--------|
| **LTX-2.3 22B Distilled FP8** (по умолч.) | 22 ГБ | ~2 мин | Отличное | t2v, i2v |
| Wan2.1 14B GGUF Q5 | 11 ГБ | ~20 мин | Очень хорошее | t2v |
| Chroma1-HD (изображения) | 20 ГБ | ~30 сек | Отличное | t2i, i2i |

Оценки скорости для AMD RX 7900 XT с ROCm 6.3+. Реальное время зависит от сложности промпта и разрешения.

## Отклоненные модели (не добавлять повторно)

| Модель | Причина | Дата |
|--------|---------|------|
| LTX-Video 2B | Качество значительно ниже 22B; макс. 720p, заметные артефакты | 2026-03 |
| LTX-2.3 22B Dev FP8 | 29 ГБ, превышает 24 ГБ VRAM даже с --lowvram | 2026-03 |
| LTX-2.3 22B GGUF Q4 | GGUF пайплайн сломан на RDNA3; OOM с VAE+text_projection | 2026-03 |

Подробные причины отклонения в заголовке `states/data/video_ai.yaml`.

## Использование CLI

```bash
# Текст-в-видео (модель по умолчанию)
gen-video "океанские волны на закате"

# Конкретная модель
gen-video -m hunyuanvideo-15 "киберпанк-город под дождем"

# Изображение-в-видео
gen-video -i ~/pic/reference.png "оживи эту сцену"

# Более высокое разрешение
gen-video -r 720p "лесной пейзаж с туманом"

# Вывод в H.264 для совместимости
gen-video --compat "таймлапс заката"

# Список установленных моделей
gen-video --list

# Пробный запуск (показать, что будет выполнено)
gen-video --dry-run "тестовый промпт"
```

## Вывод

- Видео сохраняются в `/mnt/one/video-ai/output/`
- Формат: MP4 с кодеком H.265/HEVC (аппаратное ускорение через VAAPI)
- Используйте флаг `--compat` для H.264 при отправке в браузеры/Telegram Desktop

## Устранение неполадок

### ComfyUI не запускается

Проверьте работу ROCm:

```bash
/mnt/one/video-ai/comfyui/venv/bin/python -c "import torch; print(torch.cuda.is_available())"
```

Если False, убедитесь, что пакеты ROCm установлены и GPU обнаружен:

```bash
rocminfo | grep gfx
```

### Нехватка VRAM

- Используйте Wan2.1 14B Q5 (11 ГБ) вместо LTX-2.3 (22 ГБ)
- Уменьшите разрешение: `-r 480p` вместо 720p
- Остановите Ollama и другие GPU-сервисы перед генерацией
- ComfyUI `--lowvram` выгружает в RAM, но работает медленнее

### Ошибка кодирования VAAPI

Автоматически переключается на программное кодирование. Для проверки поддержки VAAPI:

```bash
vainfo 2>&1 | grep -i hevc
```

## Архитектура

- **Salt state**: `states/video_ai.sls` — декларативное управление моделями
- **Файл данных**: `states/data/video_ai.yaml` — реестр моделей
- **Начальная настройка**: `states/scripts/video-ai-setup.sh` — ComfyUI + ROCm venv
- **Генерация**: `/mnt/one/video-ai/generate.sh` — раннер через ComfyUI API
- **CLI обертка**: `~/.local/bin/gen-video` — пользовательская команда
- **Воркфлоу**: `/mnt/one/video-ai/workflows/*.json` — ComfyUI API воркфлоу
- **Модели**: `/mnt/one/video-ai/models/<model-id>/` — загруженные чекпоинты
- **Вывод**: `/mnt/one/video-ai/output/` — сгенерированные MP4 видео
