# Настройка NotebookLM

## Обзор

NotebookLM — AI-ассистент Google для исследований. Позволяет загружать
документы (PDF, Google Docs, веб-страницы, аудио, YouTube) и задавать вопросы
с привязкой к источникам. Генерирует саммари, FAQ, учебные гайды и
**Audio Overviews** (подкаст-диалоги по загруженным материалам).

Это **веб-сервис** (нет десктопного приложения, CLI или пакета в repos/AUR).

## Доступ

| Деталь | Значение |
|---|---|
| URL | `https://notebooklm.google.com` |
| Авторизация | Google-аккаунт |
| Цена | Бесплатный тариф; NotebookLM Plus через Google One AI Premium ($20/мес) |
| API | Нет (публичного API нет, март 2026) |

## Ключевые возможности

- **Привязка к источникам** — все ответы цитируют загруженные документы, снижая галлюцинации
- **Audio Overviews** — генерация подкаст-диалогов двух ведущих по материалам
- **Мульти-источники** — до 50 источников на блокнот (PDF, Docs, Slides, веб, YouTube, аудио)
- **Интерактивный Q&A** — чат с источниками, инлайн-цитаты
- **Учебные гайды / FAQ / Таймлайны** — автогенерация структурированных саммари
- **Поддержка форматов**: PDF, Google Docs/Slides, веб-URL, YouTube, аудиофайлы, текст, .md/.txt

## Ограничения

- Только веб, нет офлайн-доступа
- Нет API для программного использования
- Лимит ~500k слов на блокнот
- Audio Overviews: генерация только на английском (может суммировать не-английские источники)
- Нет экспорта сгенерированного аудио
- Данные хранятся в Google Cloud (учитывать приватность для чувствительных документов)

## Сценарии использования

### Обогащение базы знаний RAG
NotebookLM работает как **интерактивный слой исследования** перед загрузкой
документации в локальный RAG-стек (docs-rag / code-rag / LanceDB):

1. Загрузить справочные документы (Salt docs, Hyprland wiki, мануалы) в блокнот
2. Исследовать через NotebookLM, задать вопросы, выделить ключевые разделы
3. Добавить отобранные URL в `docs_sources.yaml` для постоянной локальной индексации

### Подкаст-обзоры
Генерация Audio Overviews по сложной документации перед началом реализации —
полезно для архитектурных ревью или изучения новых инструментов.

## Интеграция с локальным RAG-стеком

В проекте уже есть пайплайн загрузки документации:

| Компонент | Роль |
|---|---|
| `docs-import` | CLI для загрузки веб-страниц, man-страниц, локального markdown в LanceDB |
| `docs_sources.yaml` | Реестр источников документации (URL, глубина, теги) |
| `llama-embed` | Локальный сервер эмбеддингов (Qwen3-Embedding-8B, порт 11435) |
| `code-rag` MCP | Гибридный текст+код поиск по индексированному проекту |
| Qdrant MCP | Векторный поиск (коллекция: `salt-project`) |

**Воркфлоу**: NotebookLM (интерактивное исследование) -> выделить ценные доки ->
добавить в `docs_sources.yaml` -> `docs-import` (постоянный локальный RAG).

## Варианты доступа с десктопа

Нативного приложения нет, ближайшие варианты:

### Вариант A: Закреплённая вкладка в Zen Browser
Закрепить `notebooklm.google.com` как постоянную вкладку в Zen Browser.
Легковесно, без дополнительных зависимостей.

### Вариант B: Chromium PWA (рекомендуется для app-like опыта)
```bash
# Установить chromium (если нет)
sudo pacman -S chromium

# Запустить и установить как PWA:
# 1. Открыть https://notebooklm.google.com в Chromium
# 2. Меню (⋮) → "Установить NotebookLM..."
# 3. Создаст .desktop-файл, запускается в отдельном окне
```

### Вариант C: Webapp Manager (AUR)
```bash
paru -S webapp-manager
# Создать ярлык веб-приложения для notebooklm.google.com
# Использует изоляцию профиля Firefox/Chromium
```
