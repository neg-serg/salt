# Стандарт обработки ошибок cmd.run

Этот документ определяет обязательный паттерн обработки ошибок для всех состояний `cmd.run` и `cmd.script` в репозитории. Соответствие проверяется автоматически тестом `tests/test_cmdrun_audit.py`.

## Обязательные элементы

Каждое состояние `cmd.run` или `cmd.script` ДОЛЖНО включать:

### 1. Директива оболочки

Явно указывайте интерпретатор. Никогда не полагайтесь на системный умолчание.

```yaml
my_state:
  cmd.run:
    - name: |
        echo "hello"
    - shell: /bin/bash    # или /bin/zsh
```

### 2. Защита идемпотентности

Каждая команда ДОЛЖНА иметь хотя бы одну защиту от повторного выполнения (Принцип конституции I):

- `creates:` — когда команда создаёт известный файл
- `unless:` — когда результат — изменение состояния системы
- `onlyif:` — для условного пропуска при отсутствии зависимостей

```yaml
install_tool:
  cmd.run:
    - name: curl -fsSL https://example.com/tool | bash
    - creates: /usr/local/bin/tool
```

### 3. Обработка ошибок

Многострочные команды ДОЛЖНЫ начинаться с `set -eo pipefail`:

```yaml
build_something:
  cmd.run:
    - name: |
        set -eo pipefail
        cd /tmp
        make && make install
    - shell: /bin/bash
    - creates: /usr/local/bin/something
```

Однострочные команды наследуют обработку ошибок оболочки и не требуют явного `set -eo pipefail`.

### 4. Аннотация исключения

Если команда обоснованно требует нестандартной обработки ошибок (например, grep возвращает 1 при отсутствии совпадения), добавьте аннотацию в поле `name`:

```yaml
check_if_present:
  cmd.run:
    - name: |
        # cmdrun:exception grep возвращает 1 когда паттерн не найден
        grep -q "pattern" /etc/config || echo "not found"
    - shell: /bin/bash
    - unless: grep -q "pattern" /etc/config
```

Аннотация `# cmdrun:exception <причина>` указывает тесту аудита пропустить проверку обработки ошибок для этого состояния.

## Уровни соответствия

Тест аудита (`tests/test_cmdrun_audit.py`) проверяет три требования для каждого состояния:

| Требование | Проверка | Серьёзность |
|------------|----------|-------------|
| Директива оболочки | Ключ `shell:` присутствует | Ошибка |
| Защита идемпотентности | `creates:`, `unless:` или `onlyif:` присутствует | Ошибка |
| Обработка ошибок | `set -eo pipefail` в многострочных командах (или `cmdrun:exception`) | Предупреждение |

## Область применения

Этот стандарт применяется ко всем состояниям `cmd.run` и `cmd.script`, включая сгенерированные макросами. Макро-состояния проверяются отдельно тестом `tests/test_macro_idempotency.py`.

Фактический рефакторинг несоответствующих состояний отслеживается в спецификации 051.
