# Лучшие практики Salt — реализовано

Этот документ описывает аудит лучших практик Salt и реализованные улучшения.

## Контекст

После исследования современных практик Salt (2024–2025) мы выявили разрывы между нашей настройкой и отраслевыми стандартами, ранжировали их по влиянию и трудоёмкости, затем реализовали топ-6.

**Уже было хорошо до аудита**: YAML-архитектура на основе данных, DRY-макросы (6 Jinja2 файлов), проверки идемпотентности, паттерны сетевой устойчивости, секреты через gopass, 7 кастомных линтеров, рендеринг матрицы фич, двуязычная документация.

**Отклонено**: salt-lint (наш `lint-jinja.py` на 1178 строк уже значительно превосходит возможности salt-lint).

## Реализованные улучшения

### 1. Pre/Post-apply снапшоты Btrfs

Каждый `just apply` автоматически создаёт пару снапшотов snapper pre/post, обеспечивая мгновенный откат.

- **Как**: `salt-apply.sh` вызывает `snapper create --type pre/post` вокруг выполнения состояний
- **Откат**: `just rollback` возвращает к последнему pre-apply снапшоту
- **Грациозно**: Пропускается если snapper недоступен

### 2. Автоматическое обнаружение дрифта

Ежедневный таймер systemd сравнивает объявленные пакеты с установленными и отправляет уведомление на рабочий стол при обнаружении дрифта.

- **Таймер**: `salt-drift.timer` (ежедневно, persistent)
- **Сервис**: `salt-drift.service` запускает `drift-notify.sh`
- **Уведомление**: `notify-send` при обнаружении дрифта
- **Логи**: `logs/drift-YYYY-MM-DD.log`
- **По запросу**: `just pkg-drift`

### 3. Визуализация графа зависимостей

Генерация визуального графа всех зависимостей состояний (include/require/watch/onchanges).

- **Команда**: `just dep-graph` (открывает SVG), `just dep-graph --format text` (дерево)
- **Вывод**: формат DOT, SVG через graphviz, или текстовое дерево
- **Обнаружение циклов**: Предупреждает о циклических зависимостях
- **Скрипт**: `scripts/dep-graph.py`

### 4. Контейнерные дымовые тесты

Тесты на основе Podman, применяющие состояния в изоляции и проверяющие результаты.

- **Команда**: `just smoke-test`
- **Образ**: `archlinux:latest`
- **Тесты**: Валидация рендеринга всех 36+ состояний, выполнение безопасных состояний, проверки
- **Скрипт**: `tests/smoke-test.sh`

### 5. Тренды профилирования состояний

Расширенный профайлер состояний с анализом трендов по нескольким логам применения.

- **Тренд**: `just profile-trend` — min/max/avg/latest длительность по состоянию
- **Сравнение**: `just profile-compare LOG1 LOG2` — выделяет регрессии (>20% медленнее)
- **Скрипт**: `scripts/state-profiler.py` (расширен)

### 6. Единая проверка здоровья

Одна команда для проверки всех управляемых сервисов.

- **Команда**: `just health`
- **Проверяет**: Системные сервисы, пользовательские сервисы, HTTP healthcheck эндпоинты
- **Вывод**: Цветная таблица, `--json`, или `--quiet` (только код возврата)
- **Скрипт**: `scripts/health-check.sh`

## Новые рецепты Justfile

| Рецепт | Описание |
|--------|----------|
| `rollback` | Откат к последнему pre-apply снапшоту |
| `dep-graph` | Генерация графа зависимостей состояний |
| `smoke-test` | Контейнерные дымовые тесты |
| `profile-trend` | Тренды длительности состояний по логам |
| `profile-compare LOG1 LOG2` | Сравнение двух логов применения |
| `health` | Проверка здоровья всех управляемых сервисов |
