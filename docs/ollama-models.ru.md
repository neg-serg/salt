# Ollama: выбор моделей

## Источник истины

Модели определены в `states/data/ollama.yaml`. Этот файл — единственный источник списка моделей; добавление и удаление производится только в нём.

## Критерий отбора

Каждая модель должна занимать **уникальную нишу** — хотя бы по одному параметру: семья обучения, архитектура (MoE vs Dense), окно контекста, размерный класс или специализация (vision, reasoning, code, uncensored). Если модель из той же семьи строго лучше при сходных ресурсах, более слабая считается дубликатом и должна быть удалена.

## Добавление и удаление моделей

1. Проверьте, не занята ли ниша существующей моделью.
2. Если новая модель вытесняет существующую — замените её, оставив закомментированную строку в `ollama.yaml` с пометкой о замене.
3. Добавьте inline-комментарий в `ollama.yaml`, объясняющий уникальную нишу.

## Хранилище

Модели хранятся на `/mnt/one/ollama/models`. Актуальные размеры — `ollama list`; ожидайте ~300GB+ при полном составе.

## Применение изменений

```bash
just apply ollama
```
