# Ollama: руководство по выбору моделей

## Обзор

Модели определены в `states/data/ollama.yaml` и скачиваются автоматически
через `ollama.sls`. Каждая модель должна занимать **уникальную нишу** —
если модель B из той же семьи строго лучше модели A при сходных ресурсах,
модель A — дубликат и должна быть удалена.

## Критерии отбора

Модель заслуживает место в списке, если уникальна **хотя бы по одному** из параметров:

| Параметр | Пример |
|----------|--------|
| Семья обучения | Qwen vs Mistral vs BigCode vs Google — разные данные, разные сильные стороны |
| Архитектура | MoE (разреженная активация) vs Dense — разные компромиссы скорость/качество |
| Окно контекста | 16K vs 128K vs 256K — сколько кода помещается в один запрос |
| Покрытие языков | 17 языков vs 600+ — важно для редких/нишевых языков |
| Размерный класс | ~4GB (быстрый локальный) vs ~19GB (качество) — разные уровни железа |
| Специализация | FIM completion vs agentic coding vs общий reasoning |

### Почему именно эти

Каждая code-модель имеет уникальную комбинацию семья + архитектура + контекст,
которую не покрывает ни одна другая модель в списке:

- **qwen2.5-coder:7b-instruct-q6_K** — единственный лёгкий (~6GB) code model; ничто другое не работает так быстро с качеством Q6_K
- **qwen3-coder:30b** — MoE с всего 3.3B активных параметров: скорость близка к 7B при качестве 30B; 256K контекст — самый большой в списке
- **deepcoder:14b** — dense 14B с 128K контекстом; заполняет среднеразмерный dense слот между 7B и 24B
- **devstral:24b** — семья Mistral (французская лаборатория, иная методология обучения); топ SWE-bench, agentic coding

## Добавление новой модели

Перед добавлением модели проверьте:

1. **Ниша уже занята?** Проверьте таблицу выше — если существующая
   модель покрывает ту же семью + размерный класс + архитектуру, новая модель
   скорее всего дубликат.
2. **Вытесняет ли она существующую?** Если новая модель из той же
   семьи строго лучше, замените старую и перенесите её в раздел
   «Исключённые».
3. **Добавьте комментарий** в `ollama.yaml` в той же строке, объясняющий
   уникальную нишу (семья, архитектура, контекст, специализация).

## Хранилище

Модели хранятся на `/mnt/one/ollama/models`. Актуальные размеры — см.
`ollama list`. Общий объём зависит от состава; ожидайте ~300GB+ со всеми
моделями включая 70B и 235B варианты.
