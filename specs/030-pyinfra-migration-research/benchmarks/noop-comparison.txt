# No-Change (Idempotent) Apply Comparison: Salt vs pyinfra
#
# ┌─────────────────┬───────────┬────────────┬──────────────────────────┐
# │ Phase           │ Salt      │ pyinfra    │ Notes                    │
# ├─────────────────┼───────────┼────────────┼──────────────────────────┤
# │ Startup/imports │ 5-6s      │ 1-2s       │ Salt loads grains,       │
# │                 │           │            │ modules; pyinfra just    │
# │                 │           │            │ imports Python           │
# ├─────────────────┼───────────┼────────────┼──────────────────────────┤
# │ State compile   │ 2-3s      │ 0s         │ Salt renders Jinja+YAML; │
# │                 │           │            │ pyinfra has no template  │
# │                 │           │            │ rendering step           │
# ├─────────────────┼───────────┼────────────┼──────────────────────────┤
# │ State execution │ 4.3s      │ ~4-5s      │ Both run same checks:    │
# │                 │           │            │ file exists, service     │
# │                 │           │            │ status, package queries  │
# │                 │           │            │ pyinfra slightly slower  │
# │                 │           │            │ due to subprocess.Popen  │
# │                 │           │            │ overhead per operation   │
# ├─────────────────┼───────────┼────────────┼──────────────────────────┤
# │ TOTAL           │ ~12s      │ ~6-7s      │ pyinfra ~5-6s faster     │
# └─────────────────┴───────────┴────────────┴──────────────────────────┘
#
# NOTE: pyinfra times are ESTIMATED (not empirically measured) because:
# 1. Running pyinfra @local for 566 states requires porting all states first
# 2. The PoC port (poc/installers.py) covers only 1 of 36 state files
# 3. The theoretical analysis is sufficient: both tools execute the same
#    external commands (pacman -Q, systemctl is-active, test -f, etc.)
#
# VERDICT:
# - Idempotent apply: pyinfra is ~5-6s faster (12s → 6-7s)
# - Fresh apply with downloads: ~0% difference (both 5-30 min)
# - Fresh apply with parallel downloads: pyinfra is SLOWER (+30 min
#   regression due to lost parallel: True on ollama/video_ai)
#
# The 5-6s saving on idempotent applies does NOT justify 3-5 weeks of
# migration effort and the loss of parallel downloads.
