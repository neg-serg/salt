## Implementation Phases

### Phase 1: Lint Infrastructure (P3 foundation, built first)

Extend `scripts/lint-jinja.py` with:
1. **Idempotency guard check** (`check_idempotency_guards`): detect `cmd.run`/`cmd.script` without `creates:`, `unless:`, `onlyif:`, or `onchanges:` in rendered YAML
2. **Inline suppression** (`# salt-lint: disable=<rule>`): parse suppression comments from source, apply to rendered state IDs
3. **Promote network resilience** from warnings to errors (configurable)

*Rationale*: Building the lint first lets us validate all subsequent changes automatically. Every fix in Phase 2-5 should reduce lint violations toward zero.

### Phase 2: Macro Extensions (P2)

1. **`npm_pkg` version parameter**: Add `version=''` param; when set, use version stamp file instead of `creates:` guard
2. **`paru_install` version parameter**: Same pattern as npm_pkg
3. **`user_service_with_unit()` macro**: Combine `user_service_file()` + `user_service_enable()` into single call with internal require chain
4. **Update callers**: Refactor states that manually chain service file + enable to use the new combined macro (where appropriate — batch-enable patterns keep separate calls)

### Phase 3: Idempotency & Network Resilience Fixes (P1)

File-by-file audit and fix:
1. Add missing `creates:`/`unless:`/`onlyif:` guards to all `cmd.run`/`cmd.script` states
2. Add `retry: {attempts: retry_attempts, interval: retry_interval}` to all network states
3. Add `parallel: True` to independent network states (respecting existing `process_count_max: 16` limit)
4. Add suppression comments where intentional (e.g., `onchanges`-triggered states)

### Phase 4: Explicit Requisites (P2)

1. Add missing `require:` between dependent states (mount → dir, service → config, model pull → health check)
2. Strengthen `watch:`/`onchanges:` chains where implicit ordering was relied upon
3. Validate with `check_require_resolve` in lint script

### Phase 5: Data Extraction & File Splits (P2/P3)

1. **Data extraction**: Create `data/desktop.yaml` with Hyprland packages; update `desktop.sls` to use `import_yaml`
2. **File splits**:
   - `installers.sls` → `installers.sls` + `installers_mpv.sls`
   - `services.sls` → `services.sls` + `services_bitcoind.sls`
   - `monitoring.sls` → `monitoring.sls` + `monitoring_loki.sls`
3. **Include list**: Update `system_description.sls` with new module names
4. **Verify**: `just` + full apply to confirm identical system state

### Phase 6: Validation & Baseline (all priorities)

1. Run `python3 scripts/lint-jinja.py` — expect zero errors
2. Run `just` — expect clean render
3. Run `salt-call --local state.apply` twice — expect zero changes on second run
4. Measure no-change apply time vs baseline for SC-002
